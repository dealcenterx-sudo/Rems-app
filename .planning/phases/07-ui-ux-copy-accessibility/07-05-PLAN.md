<objective>
Standardize the three shared interaction primitives so the rest of the phase can adopt them:
(1) `LoadingButton` gains a present-tense pending-label prop (D-14) and its byte-identical hex is
tokenized (D-15/RC-01); (2) `ConfirmModal` gains focus trapping via `useFocusTrap` (D-19) plus a
loading state on its confirm button, and its destructive copy contract is verified against handler
behavior (COPY-03/D-07); (3) `Toast.error` stays a string API but call sites compose the central
map's `{ message, recovery }` (D-13) — optionally via an errorMessages `toToastString` helper.

Purpose: These primitives are consumed by optimistic toggles + pending-button adoption (plan 07)
and by ConfirmModal's callers. They must be standardized before adoption.
Output: Extended Loading.js, ConfirmModal.js, Toast.js (byte-identical token migration of these
three files is done here; they are EXCLUDED from the token sweep plans 10/11 to avoid double-edit).

- ConfirmModal focus trap + `loading`/`confirming` prop swapping confirm to LoadingButton
- Toast error composition wiring (call-site `${message} ${recovery}` and/or errorMessages.toToastString)
</objective>
