# Phase 7 — UI Design Contract

> Visual and interaction contract for the UI/UX, Copy & Accessibility polish phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase POLISHES existing REMS surfaces. It introduces **no new design language and no UI kit**. Every value below is derived from the existing token system in `src/App.css :root` and the 21 locked decisions in `07-CONTEXT.md` (D-01…D-21). Where a value must change (contrast, focus ring), the change happens **at the token level** so it propagates everywhere.

## Spacing Scale

Source of truth = existing `--space-*` tokens. All are multiples of 4. **Use tokens, never raw px.**

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline chip padding |
| `--space-2` | 8px | Compact element spacing, `LoadingButton` spinner↔label gap |
| `--space-3` | 12px | Empty-state message top margin, action-row gap, card inner gaps |
| `--space-4` | 16px | Default element / form-field spacing |
| `--space-5` | 20px | Empty-state actions top margin, section rhythm |
| `--space-6` | 24px | Section padding |
| `--space-8` | 32px | Major section breaks |

**Exceptions (existing, do not "fix" in this phase):** `.page-content` padding uses `clamp(16px, 2.4vw, 30px)`; `.section` uses `22px`. These predate the token scale and are out of scope for the token migration (D-16 leaves layout/responsive values alone). New skeleton/empty-state work MUST use the tokens above and MUST NOT introduce new raw-px spacing.

## Typography

Existing 8-step scale (`--font-xs` 11px … `--font-3xl` 28px) is **pre-existing and locked** — this phase does not redesign it. `11px` is the hard type floor (`--font-xs`). The roles below are the ones this phase's new/edited surfaces MUST use.

| Role | Token / Size | Weight | Line Height | Where |
|------|--------------|--------|-------------|-------|
| Body | `--font-base` 14px | 400 | 1.45 (body default) | Empty-state message, toast body, error copy |
| Label / meta | `--font-sm` 12px / `--font-xs` 11px | 500–600 | 1.4 | Eyebrows, button labels, KPI captions |
| Section heading | `--font-xl` 20px | 600 | 1.2 | Card/section titles |
| Page title | `clamp(18px, 3vw, 24px)` (`page-state-title`) | 800 | 1.2 | Empty-state title, page headers |

**Weights:** primary pair is **400 (body/regular)** and **600 (semibold)**. `800` is an existing exception reserved for page/empty-state titles (`.page-state-title`, `.page-header h2`) — preserve it, do not spread it to new elements. Do not introduce weights outside {400, 500, 600, 800}.

**Line heights:** body/paragraph copy 1.45; all headings/titles 1.2. Skeleton text-line shapes MUST reserve height matching the text they replace (see no-CLS rule, D-10).

## Color

REMS is a dark, near-black brand. The 60/30/10 split maps to the surface tiers:

| Role | Token(s) | Value | Usage |
|------|----------|-------|-------|
| Dominant (~60%) | `--surface-0` / `--surface-1` | `#000000` / `#0a0a0a` | Page background, app shell, base canvas |
| Secondary (~30%) | `--surface-2` / `--surface-3` | `#0f0f0f` / `#141414` | Cards, sections, sidebar, nav, table rows, skeleton base |
| Accent (~10%) | `--accent` (+ `--accent-strong`/`--accent-soft`/`--accent-border`) | `#00ff88` | **Reserved list below** |
| Destructive | `--danger` (+ `--danger-hover`/`--danger-soft`) | `#ff3333` | Destructive actions + error state only |

**Accent (`#00ff88`) reserved for — explicit list (never "all interactive elements"):**
1. Primary CTA buttons (`.btn-primary`) and the single quiet empty-state action.
2. The keyboard focus indicator (focus ring) — see A11y token targets.
3. Active/selected navigation + subnav state.
4. Positive / success status (success toasts, "complete"/"active" status pills) — **always paired with a text or icon cue, never color alone (D-20)**.
5. Key-metric emphasis on Home KPI tiles.

**Text tiers (locked names, values may change in Pass 2 only):** `--text-primary` `#f1f1f1`, `--text-secondary` `#cfcfcf`, `--text-muted` `#8a8a8a`, `--text-faint` `#757575` (minimum legible tier).

**Semantic (non-brand):** `--info` `#0088ff` (informational only — NOT a focus color; see D-18), `--warning` `#ffaa00`, `--danger` `#ff3333`.

## Copywriting Contract

**Voice (D-01):** minimal, factual, enterprise-austere. Sentence case. Verb + object buttons. One term per concept. No marketing warmth, no personality. A one-page copy standard (`COPY-01`) is authored first, then applied to high-traffic surfaces (D-06: Home, Deals, CRM/Leads, Contacts, Properties, Tasks, Documents, Settings, auth, confirmation dialogs).

### Empty states (built on `PageState`, D-04)

Two variants on the ~6 primary list surfaces (Deals, CRM/Leads, Contacts, Properties, Tasks, Documents — D-02). Secondary surfaces get one generic empty state.

| Variant | `title` | `message` | `actions` |
|---------|---------|-----------|-----------|
| First-use (no data yet) | Noun, e.g. `No deals yet` | One factual line stating what appears here + how to add the first item | One quiet primary action, e.g. `New deal` (understated, single action — D-03) |
| No-results (filtered to zero) | `No matches` | One factual line: nothing matches the current filters | Quiet `Clear filters` affordance (D-03) — never identical to first-use |
| Error (load failed) | `Couldn't load {noun}` | Human message + recovery from central map (D-05) | `Try again` where a retry handler exists |

Concrete per-surface title strings are authored during planning against the copy standard; the pattern above is the contract. Empty states set `PageState` `tone="empty"` (or `"error"` for load failures) and rely on its existing `aria-live` (`polite`/`assertive`).

### Error copy (central map — `src/utils/errorMessages.js`, D-05)

- **Single source of truth:** `code → { message, recovery }`. Generalizes the existing `friendlyAuthError` in `LoginPage.js`. Unit-testable. Covers the Firebase/API codes the live flows actually surface (Claude's discretion on the exact code list).
- **Format contract:** `message` = one plain sentence naming the problem (no error codes, no stack, no security detail leaked). `recovery` = one imperative next step. Example shape: `{ message: "That email is already in use.", recovery: "Sign in instead, or use a different email." }`.
- **Surfacing:** rendered via the existing `Toast` system (error tone) and in `PageState` error states. Optimistic-write failures show `message` + `recovery` in an **error toast** with **no Retry action** (D-13).

### Destructive confirmations (COPY-03, via `ConfirmModal`)

- Copy MUST be **verified against actual handler behavior before rewriting (D-07)** — never describe an outcome the handler does not perform (e.g. don't say "permanently deleted" if it soft-archives).
- Pattern: state the object + the irreversible consequence in one line; confirm button uses a present-tense verb+object label that becomes the pending label when in flight (see `LoadingButton`).

| Element | Copy contract |
|---------|---------------|
| Primary CTA (per surface) | Verb + object, sentence case (e.g. `New deal`, `Add contact`, `Save changes`) |
| Empty-state heading | Terse noun-phrase (`No deals yet` / `No matches`) |
| Empty-state body | One factual sentence + next step |
| Error state | `{plain problem sentence}` + `{one imperative recovery step}` from `errorMessages.js` |
| Destructive confirmation | `{Delete X}?` + one-line real consequence, verified against handler (D-07) |

## Component & Interaction Contract

This phase is interaction- and token-heavy rather than new-screen layout. The following component specs are part of the design contract the executor implements.

### `<Skeleton>` primitive (NEW — `src/components/`, D-08/09/10)

One reusable shimmer primitive plus composed shapes; pages assemble them to mirror their own layout with **zero layout shift**.

| Aspect | Contract |
|--------|----------|
| Base fill | `var(--surface-2)` (`#0f0f0f`) |
| Shimmer sweep | Moving highlight lighter than base. Recommended: `linear-gradient(90deg, var(--surface-2) 25%, #1a1a1a 50%, var(--surface-2) 75%)`, `background-size: 200% 100%`. `#1a1a1a` is a documented one-off highlight (between `--surface-3` and `--border-strong`); if promoted, add token `--skeleton-highlight: #1a1a1a` in Pass 1 rather than inlining. |
| Animation | `shimmer` keyframe (already in App.css), duration **1.2s**, `linear`, infinite. Border-radius matches the shape it stands in for (`--radius-sm`/`--radius-md`). |
| Composed shapes | `text-line`, `card`, `table-row`, `KPI tile` — each sized to reserve the exact height/width of the real content it replaces (no CLS on swap-in, D-10). |
| Reduced motion | Under `@media (prefers-reduced-motion: reduce)`: disable the sweep animation, render a static `--surface-2` fill. |
| Accessibility | Skeleton container is decorative: `aria-hidden="true"` (screen readers hear the page's existing `aria-live`/`role="status"` loading affordance, not the shimmer). |

**Delay-then-show (D-09):** render the skeleton only after loading exceeds a threshold in the **300–500ms** band. **Contract value: 400ms.** Sub-threshold loads go straight to content with no skeleton flash. Applies to skeleton pages: Home, Deals, CRM, Contacts, Properties, Tasks, Documents (D-10).

**Home KPIs (UI-03, D-11):** silent SWR — render cached KPIs instantly, refetch in background, swap in new numbers when ready, **no visible refresh indicator, no skeleton on cached render**. Extend the existing HomePage localStorage KPI cache (30s TTL); do not add a new store. Skeleton for KPI tiles is only for the true cold/no-cache first load.

### `LoadingButton` standardization (`src/components/Loading.js`, D-14/15)

Standardize the existing component as the single pending-state pattern for every submit/destructive button.

| State | Contract |
|-------|----------|
| Idle | Renders `children` label; `--accent` fill for primary, `--danger` for destructive. |
| Pending | `disabled`; inline spinner (16px) + gap `--space-2`; label switches to **present-tense verb** (`Saving…`, `Deleting…`, `Sending…`) — replace the current generic `Loading...`. |
| Token fix (D-15) | Replace hardcoded hex in this file with tokens: default spinner color `#ffffff` → `var(--text-primary)`; `LoadingSpinner` default `#00ff88` → `var(--accent)`. This is part of the UI-05 migration, not a separate visual change. |
| Disabled affordance | `opacity: 0.6`, `cursor: not-allowed` (existing) — keep. |

### Optimistic toggles (UI-06, D-12/13)

- **Scope: toggles only** — task complete/incomplete, task/deal status changes. All other mutations use normal pending→confirm. Inline field edits and drag/reorder are explicitly **not** optimistic this phase.
- On success: no extra chrome (the toggle already reflects the new state).
- On failure: **silently revert** the control to prior state + show one **error toast** (`message` + `recovery` from `errorMessages.js`). No inline per-control error, no Retry.

## Accessibility & Token Targets (A11Y-01/02/03, D-16…D-21)

### Focus indicator — reconciliation (D-18) — **critical existing bug**

`src/App.css` currently defines `--focus-ring` **twice** and the app's primary `:focus-visible` selector actually renders an **off-brand blue** ring:

- Line 73: `--focus-ring: 0 0 0 3px rgba(0, 255, 136, 0.25)` (green)
- Line ~4347: `--focus-ring: 0 0 0 3px rgba(0, 255, 136, 0.18)` (green, fainter — this later definition wins)
- Line ~4351: `--shadow-focus: 0 0 0 3px rgba(0, 136, 255, 0.25)` (**blue**), consumed by the global `:focus-visible` block (~line 4494) covering buttons/links/inputs/nav.

**Contract (Pass 2):**
1. Collapse to **one** `--focus-ring` definition (delete the duplicate).
2. **Delete `--shadow-focus`** and repoint every `:focus-visible` rule that used it to `--focus-ring`, so the focus indicator is brand-green everywhere.
3. Set the unified ring opaque enough to meet the **3:1** non-text UI contrast for the indicator. Faint `0.18` is too weak. **Target value: `--focus-ring: 0 0 0 3px rgba(0, 255, 136, 0.45)`** (≈3.5:1 composited over `--surface-1`; `#00ff88` itself is ~15.7:1 vs `#000`, so a solid `2px` `--accent` outline is an acceptable alternative). Keyboard-only via `:focus-visible`; visible focus MUST never be removed without an equivalent replacement.

### Text contrast — WCAG 2.2 AA (4.5:1 normal text), token-level (D-18, Pass 2)

Computed against `--surface-3` (`#141414`, the lightest common content surface):

| Token | Current | Contrast on `--surface-3` | Verdict | Pass-2 target |
|-------|---------|---------------------------|---------|---------------|
| `--text-primary` `#f1f1f1` | — | ~13.6:1 | PASS | keep |
| `--text-secondary` `#cfcfcf` | — | ~9.6:1 | PASS | keep |
| `--text-muted` `#8a8a8a` | — | ~5.3:1 | PASS | keep (do not darken) |
| `--text-faint` `#757575` | minimum tier | **~4.0:1** | **FAIL (normal text)** | **`#7f7f7f`** → ~4.6:1 on `--surface-3` |

`--text-faint` is the only text tier failing AA for normal-size text and is used at 14px (e.g. `LoadingOverlay`). Bump it to **`#7f7f7f`** (or lighter) in Pass 2. Verify any tier used on hover/gradient surfaces lighter than `#141414` still clears 4.5:1 after migration. Pass-2 token value changes are a **separate commit** from Pass 1 (D-17).

### Two-pass token migration gate (UI-05, D-16/17)

- **Pass 1 (byte-identical):** map each of the ~1,200 hex values to a token whose value is **byte-identical**, or add an exact-match token. Computed styles unchanged by construction. Add tokens for recurring values lacking one; leave truly one-off decorative/chart colors inline **but documented**. Gate: green `npm run build` + `npm run lint` + diff review. **No token value changes in Pass 1.**
- **Pass 2 (contrast):** adjust the specific token *values* above (`--text-faint`, `--focus-ring`, delete `--shadow-focus`) to meet 4.5:1 / 3:1 while preserving brand direction. Separate commit(s).

### Keyboard, focus trap, semantic HTML (A11Y-02, D-19/20)

| Item | Contract |
|------|----------|
| `useFocusTrap` hook (NEW, `src/utils/`, D-19) | Focus first interactive element on open, cycle Tab/Shift+Tab within, restore focus to the invoking element on close. **No new dependency.** Pairs with existing `useEscapeKey`. |
| Coverage | Apply to `ConfirmModal` and the **major** modals. Exhaustive per-drawer/tab coverage is explicitly deferred (D-19). |
| Modal behavior (D-20) | Escape-close + initial-focus-on-open on modals; all interactive elements keyboard-reachable; semantic HTML on major pages. |
| Never color-alone (D-20) | Any status conveyed by color (accent = active/success, `--danger` = error, `--warning`) MUST also carry a text label or icon cue. |

### Lint gate (A11Y-03, D-21)

Enable `eslint-plugin-jsx-a11y` **recommended ruleset at error level** in `package.json` `eslintConfig` (CRA `react-app` extends). Fix all violations **in the same change** that flips the rule to error, so CI lint (runs on every push) stays green.
