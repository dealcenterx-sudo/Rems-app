  <behavior>
    - On activate: focuses the first tabbable element inside the container ref.
    - Tab from the last tabbable wraps to the first; Shift+Tab from the first wraps to the last.
    - On deactivate/unmount: focus is restored to the element that was focused before activation (the invoker).
    - When inactive (active=false): the hook adds no listeners and does not move focus.
  </behavior>
  <action>
    Create `src/utils/useFocusTrap.js` mirroring the useEscapeKey structure exactly: `useEffect`
    with an `active` guard and `return undefined` early-exit, add/remove keydown listener in the
    effect, cleanup in the returned function, JSDoc one-liner, `export default`. Signature
    `useFocusTrap(containerRef, active = true)`. On activate capture `document.activeElement`, then
    focus the first tabbable inside the container (query focusable selectors via
    `container.querySelectorAll` — buttons, links with href, inputs/selects/textareas not disabled,
    `[tabindex]:not([tabindex="-1"])`). On Tab/Shift+Tab keydown, cycle within the container
    (wrap first↔last). On cleanup, restore focus to the captured invoker. No new dependency. Write
    `useFocusTrap.test.js` with RTL rendering a container of focusable elements and asserting the
    four behaviors above.
  </action>
