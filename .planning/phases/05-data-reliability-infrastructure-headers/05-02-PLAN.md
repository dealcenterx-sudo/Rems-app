<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Vercel edge (CSP header) | The CSP Report-Only header instructs the browser which origins may load scripts/frames/connections; an over-wide allowlist weakens the future enforced policy |
| CRA build → deployed bundle shape | INLINE_RUNTIME_CHUNK controls whether the webpack runtime is inline (needs 'unsafe-inline') or external ('self'-servable) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05B-01 | Tampering | inline runtime script under script-src 'self' | high | mitigate | INLINE_RUNTIME_CHUNK=false emits an external runtime chunk so no inline <script> is needed — removes the 'unsafe-inline' XSS blast-radius before Phase 8 enforcement |
| T-05B-02 | Elevation of Privilege | CSP allowlist over-widening | medium | mitigate | Only the three known Google-sign-in / Sentry-region hosts are added (wildcards scoped to *.firebaseapp.com / *.ingest.us.sentry.io); header stays Report-Only so nothing is enforced yet; soak surfaces the rest before tightening in Phase 8 |
| T-05B-03 | Information Disclosure | secrets in config/headers | high | mitigate | Only non-secret host allowlist strings and a non-secret build flag are added; no DSN, key, or secret value is written to package.json or vercel.json |
| T-05B-SC | Tampering | npm/pip/cargo installs | n/a | accept | This plan installs NO packages (RESEARCH Package Legitimacy Audit: zero new installs) — no supply-chain surface |
