<objective>
Land the local, emulator-verifiable half of SEC-04: remove the `match /{document=**}` admin write catch-all from `firestore.rules` (the audit-log tamper vector), and flip the Phase-2 `activity_log` characterization test's two admin assertions from `assertSucceeds` to `assertFails` so the suite now ASSERTS the append-only-against-admin guarantee. `isAdmin()` is already role-only in git (the SEC-04 email-fallback code half shipped in commit dd6364a) — no email literal edit is needed here.

Purpose: Close the HIGH audit-integrity gap found in Phase 2 — the broad catch-all (`allow read, write: if isAdmin()`) is OR'd with `activity_log`'s `allow update, delete: if false`, and in Firestore a deny cannot override an allow in another match, so an admin can currently edit/delete audit entries. Removing the catch-all makes `if false` the only matching write rule. This is the ONLY new rules change this phase makes; the file must stay emulator-green because it is the exact artifact the operator pastes into the Console in 06-03.
Output: an edited `firestore.rules` (catch-all removed, role-only isAdmin() untouched) and a flipped, green `tests/rules/firestore.rules.test.js`.
</objective>
