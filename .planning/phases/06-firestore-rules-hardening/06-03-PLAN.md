Close the LIVE half of SEC-04 — the highest-risk work of the milestone — via three ordered steps that only the operator can perform: (1) the LOCKOUT GATE — verify production `users/{adminUid}.role == 'admin'` for every admin account and inspect the live Console rules; (2) the STAGED CONSOLE PUBLISH of the hardened rules from 06-01 (optional additive `role OR email` bridge if the live rules are still email-based, then the subtractive role-only + catch-all-removed publish), with the emulator suite green before and after and a two-account smoke after each publish; (3) record the Phase 6 outcome in the running changelog.

Publishing role-only rules while production `users/{adminUid}.role` is absent would deny the admin everything — hence the lockout gate MUST precede any publish, and the two-account smoke MUST follow each publish.
Output: a GO/NO-GO lockout-gate confirmation, a staged-publish + two-account-smoke result (or an explicit deferral pending Console access), and a Phase 6 changelog entry.

Expected terminal state: human_needed — the code + emulator-green + doc (06-01/06-02) are complete and locally verified; the live publish and smoke require the operator and gate final closure.
